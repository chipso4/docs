## `GET /v1/entitlement-information`

The primary entitlement check used before filing a warranty claim. Returns detailed warranty coverage tiers, applicable labor rates, homeowner information, and purchaser details.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `SerialNumber` | string | **Yes** | The unit's serial number |
| `ModelNumber` | string | No | Model number (required if serial maps to multiple models) |
| `RegisteredOwnerLastName` | string | No | Last name of registered owner (for ownership verification) |
| `StateCode` | string | No | State code of installation (may affect coverage terms) |
| `CustomerId` | string | No | Distributor/customer account ID |
| `InstallationType` | string | No | `"Residential"`, `"Commercial"`, `"MultiFamily"` |

### Key response fields

| Field | Description |
|-------|-------------|
| `coverages` | Array of [`WarrantyCoverage`](/data-types/warranty) objects — one per coverage tier. This is the most important field for claim entitlement decisions. |
| `laborRates` | Applicable [`LaborRateSchedule`](/data-types/warranty) with rate data in `extendedAttributes`. |
| `claimSubmissionDeadlineDays` | How many days from the failure date a claim must be submitted. |
| `homeowner` | Registered homeowner info — pre-fills the claims form. |
| `purchaserDetails` | Distributor that sold the unit. |
