## Base URL

```
https://{your-host}/v1
```

Replace `{your-host}` with your middleware API's hostname.

## Authentication

All endpoints require an OAuth2 bearer token. See [Authentication](/getting-started/authentication) for details.

```
Authorization: Bearer {access_token}
```

## Endpoint summary

Your middleware must implement the following 23 endpoints. Click any endpoint for the full specification.

### Unit & product data

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | [`/v1/validate-unit`](/api-reference/unit-data/validate-unit) | Validate serial number, return model(s) |
| GET | [`/v1/units/{serial_number}`](/api-reference/unit-data/get-unit) | Full unit record |
| GET | [`/v1/entitlement-information`](/api-reference/unit-data/entitlement-information) | Warranty entitlement with coverage tiers |
| GET | [`/v1/units/{serial_number}/warranty-coverages`](/api-reference/unit-data/warranty-coverages) | Coverage tiers only |
| GET | [`/v1/units/{serial_number}/parts`](/api-reference/unit-data/unit-parts) | BOM with specs and images |
| GET | [`/v1/units/{serial_number}/distributor`](/api-reference/unit-data/unit-distributor) | Distributor that sold the unit |

## Response conventions

- All responses are `application/json`
- Dates use ISO 8601 format (`2024-01-15` or `2024-01-15T14:30:00Z`)
- All errors follow the [standard error format](/getting-started/error-handling)
- List endpoints support [pagination](/data-types/core-objects#paginatedresponse)
- Every entity supports [`extendedAttributes`](/overview/key-concepts#the-extendedattributes-pattern) for custom fields
