## `GET /v1/import/registrations`

Bulk export existing warranty registration data from your legacy systems. Powers the one-time initial data load and ongoing incremental sync so Continuum starts with the full registration history.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `since` | ISO 8601 date | No | Only return registrations created/updated after this date (for incremental sync) |
| `page` | integer | No | Page number (default: 1) |
| `page_size` | integer | No | Results per page (default: 100, max: 1000) |

### Pagination

Continuum calls this endpoint repeatedly with increasing page numbers until all pages are consumed:

```
GET /v1/import/registrations?page=1&page_size=100
GET /v1/import/registrations?page=2&page_size=100
...
GET /v1/import/registrations?page=453&page_size=100
```

For incremental sync after the initial load:

```
GET /v1/import/registrations?since=2026-03-01T00:00:00Z&page=1&page_size=100
```

### Implementation notes

- **Include coverage tiers** in each record so Continuum can display accurate warranty status without re-calling entitlement for every imported unit.
- **Timeout:** Continuum allows up to 60 seconds for import responses (longer than the standard 30-second timeout).
- **Idempotency:** Continuum handles duplicates gracefully — the most recent data wins.
