## `POST /v1/return`

Create a new RMA. The request body is a [`CoreRma`](/data-types/core-objects#corerma) entity with [`CoreAccountLocation`](/data-types/core-objects#coreaccountlocation) composition for addresses and [`CoreRmaReturnItem`](/data-types/core-objects#corermareturniitem) for line items.

<Tabs>
  <Tab title="Standalone return">
    Used in the [return flow](/returns-hub/return-flow) when a customer is returning a non-defect product. The RMA is created in an active state and proceeds through acknowledge → receive → credit.
  </Tab>
  <Tab title="Warranty flow">
    Used in the [warranty flow](/warranty-hub/warranty-flow) to create an RMA in **DRAFT** status early in the process. The RMA sits dormant until the warranty claim is approved, then is finalized in Phase 5.
  </Tab>
</Tabs>

The request body follows the [`CoreRma`](/data-types/core-objects#corerma) schema. Required fields:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `customerId` | string | **Yes** | Customer identifier |
| `items` | [CoreRmaReturnItem[]](/data-types/core-objects#corermareturniitem) | **Yes** | At least one line item |
| `items[].partNo` | string | **Yes** | Product/part number |
| `items[].quantity` | decimal | **Yes** | Return quantity |
| `items[].unitPrice` | decimal | **Yes** | Unit price |
| `items[].unitOfMeasure` | string | **Yes** | Unit of measure |

All other fields from the `CoreRma` schema are optional. See the [core objects reference](/data-types/core-objects#corerma) for the full field listing.

### Response `201 Created`

Returns the full [`CoreRma`](/data-types/core-objects#corerma) entity as created in the ERP, including the assigned `rmaNumber` and `status`.

```json
{
  "rmaNumber": "RMA-2026-00100",
  "status": "OPEN",
  "createdDate": "2026-03-10T15:00:00Z",
  "customerId": "C-827451",
  "customerName": "Northern Manufacturing Inc."
}
```
