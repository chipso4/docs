## `POST /v1/sales-order`

Create a new sales order for a warranty replacement. The request body follows the [`CoreOrder`](/data-types/core-objects#coreorder) schema with [`CoreAccountLocation`](/data-types/core-objects#coreaccountlocation) for addresses and [`CoreOrderItem`](/data-types/core-objects#coreorderitem) for line items. The part is typically picked and handed to the customer at the counter.

The request follows the [`CoreOrder`](/data-types/core-objects#coreorder) schema. Required fields:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `customerId` | string | **Yes** | Customer identifier |
| `billTo` | [CoreAccountLocation](/data-types/core-objects#coreaccountlocation) | **Yes** | Bill-to account and address |
| `shipTo` | [CoreAccountLocation](/data-types/core-objects#coreaccountlocation) | **Yes** | Ship-to account and address |
| `items` | [CoreOrderItem[]](/data-types/core-objects#coreorderitem) | **Yes** | At least one line item |
| `items[].partNo` | string | **Yes** | Product/part number |
| `items[].quantity` | decimal | **Yes** | Quantity to order |
| `items[].unitPrice` | decimal | **Yes** | Unit price |
| `items[].unitOfMeasure` | string | **Yes** | Unit of measure |
| `paymentTermsCode` | string | **Yes** | Payment terms code |

All other fields from the `CoreOrder` schema are optional. See the [core objects reference](/data-types/core-objects#coreorder) for the full field listing.

### Response `201 Created`

Returns the full [`CoreOrder`](/data-types/core-objects#coreorder) entity as created in the ERP, including the assigned `orderNumber` and `status`. See [GET /v1/sales-order](/api-reference/orders/get-order) for the response shape.
