## `GET /v1/customers/locations`

Retrieve all addresses (locations) for a customer, regardless of type. These are customer addresses — ship-to, bill-to, and sold-to — not warehouse locations.

<Note>
**V2 equivalent:** `GET /api/btu/v2/organization/{organizationId}/locations` — returns [`CoreLocation`](/data-types/core-objects) entities with `locationType` filters.
</Note>

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `customerId` | string | **Yes** | The customer's unique identifier |
| `searchName` | string | No | Filter locations by name |
| `searchStateCode` | string | No | Filter by two-letter state code (e.g., `"TX"`) |

### Type-specific convenience endpoints

For common location lookups, use these filtered endpoints instead:

| Method | Endpoint | Filter |
|--------|----------|--------|
| GET | `/v1/customers/locations/shipto` | Ship-to addresses only |
| GET | `/v1/customers/locations/shipto/{locationId}` | Specific ship-to |
| GET | `/v1/customers/locations/billto` | Bill-to addresses only |
| GET | `/v1/customers/locations/billto/{locationId}` | Specific bill-to |
| GET | `/v1/customers/locations/soldto` | Sold-to addresses only |
| GET | `/v1/customers/locations/soldto/{locationId}` | Specific sold-to |

All type-specific endpoints accept `customerId` as a query parameter and return the same location shape.
