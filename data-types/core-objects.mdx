These are the canonical data types from `primary-data-types.json`. All API responses use these shapes. All fields are nullable unless marked required — this accommodates ERP systems that may not populate every field.

---

## DataAttribute

The extensibility mechanism for the entire API. A generic name-value pair for adding custom data to any entity.

```json
{
  "name": "TaxExempt",
  "value": "true",
  "section": "Tax"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | **Yes** | Attribute name |
| `value` | string | No | Attribute value |
| `section` | string | No | Logical grouping for UI display (e.g., `"Tax"`, `"Capacity"`, `"Electrical"`) |

<Accordion title="Recommended section names">

| Section | Example attributes |
|---------|-------------------|
| `Capacity` | CapacityGallons, CapacityBTUH, CapacityTons |
| `Electrical` | Voltage, Phase, Hertz, Amperage, FLA |
| `Efficiency` | UEF, SEER, SEER2, EER, EnergyFactor |
| `Performance` | FirstHourRating, RecoveryRate, CFM |
| `Physical` | Dimensions, Weight, MaterialType |
| `Refrigerant` | RefrigerantType, RefrigerantChargeOz |
| `Classification` | ERPMaterialGroup, ProductClass |
| `Service` | InstallationNotes, MaintenanceInterval |
| `Tax` | TaxExempt, TaxExemptNumber |

</Accordion>

## CoreAddress

Unified address value object. Used wherever physical addresses appear across all entities.

```json
{
  "address1": "1200 Industrial Blvd",
  "address2": "Suite 300",
  "address3": "Building C",
  "city": "Dallas",
  "state": "Texas",
  "stateCode": "TX",
  "zip": "75207",
  "country": "United States",
  "countryCode": "US",
  "attentionLine": "Attn: Receiving Dept"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `address1` | string | Primary street address |
| `address2` | string | Secondary line (suite, unit) |
| `address3` | string | Additional line |
| `city` | string | City |
| `state` | string | State/province full name |
| `stateCode` | string | State/province abbreviation |
| `zip` | string | Postal/zip code |
| `country` | string | Country full name |
| `countryCode` | string | ISO 3166-1 alpha-2 code (e.g., `"US"`, `"CA"`) |
| `attentionLine` | string | Attention line for the address |

## CoreContact

Standalone contact entity. Contacts are first-class citizens — not embedded fields on a location.

```json
{
  "contactId": "CTX-9012",
  "firstName": "Jane",
  "lastName": "Smith",
  "title": "Warehouse Manager",
  "email": "jane.smith@example.com",
  "phone": "214-555-0101",
  "mobile": "214-555-0102",
  "fax": "214-555-0199",
  "contactType": "Shipping",
  "isPrimary": true,
  "isActive": true,
  "address": { },
  "extendedAttributes": []
}
```

| Field | Type | Description |
|-------|------|-------------|
| `contactId` | string | Unique identifier |
| `firstName` | string | First name |
| `lastName` | string | Last name |
| `title` | string | Job title |
| `email` | string | Email address |
| `phone` | string | Primary phone |
| `mobile` | string | Mobile phone |
| `fax` | string | Fax number |
| `contactType` | string | ERP-specific classification (e.g., `"Billing"`, `"Shipping"`, `"AP"`). String by design — each ERP maps their own values. |
| `isPrimary` | boolean | Whether this is the primary contact |
| `isActive` | boolean | Whether the contact is active |
| `address` | [CoreAddress](#coreaddress) | Contact address |
| `extendedAttributes` | [DataAttribute[]](#dataattribute) | Custom attributes |
